What Makes a Lightweight Game-Playing Agent Strong

AIIDE 2026 — team brief (plan, live status & early results). Title: A Gold-Standard Study of What Makes a Lightweight Game-Playing Agent Strong.

Abstract: submitted (Jun 26) Full paper: Jul 3 Notify: Aug 7 AAAI 2-col · 9pp · double-blind · EasyChair

The paper in one breath

We build a fixed, deterministic expert for Gin Rummy and use it only as a grading yardstick (never to train). Against it we run a controlled study of what makes a small RL agent strong — algorithm, reward, curriculum, checkpointing, representation — and report honestly what helps and what doesn't. The expert ceiling (~36% win-rate for our best agent) is a clean, reproducible benchmark, and the study isolates the source of every gain. We are now adding a network-architecture axis, real imperfect-information baselines, and a second game (Leduc Hold'em with a CFR-optimal expert) to show the method generalises.

Abstract (submitted; revisable until Jul 3)

Reinforcement-learning agents for imperfect-information card games are only as strong as the opponents they train against, and practitioners rarely have a cheap, absolute yardstick for how good an agent really is. We address this by building a strong, fixed, deterministic expert for Gin Rummy that decomposes melds exact-optimally and otherwise plays principled endgame moves, and by using it only as a grading yardstick, never for training. The expert beats every agent we trained 70–99% of the time, yet gins in under 2% of games: it wins by knocking early with low deadwood. Graded against this fixed reference across more than one hundred controlled runs, we isolate what makes a lightweight agent strong: trust-region updates, a knock-first reward, a rising opponent curriculum, warm-starting, and keeping the best checkpoint each help, whereas learned state embeddings, dense step rewards, imitation learning, and a live large-language-model opponent each fail for an identifiable reason. Reward shaping cannot induce the human-intuitive but losing habit of chasing gin, and stacking the choices that help lifts a self-play champion from roughly 30% to 36% against the expert. We then test whether network architecture is the missing lever—comparing MLP width and depth, convolutional, permutation-invariant set, attention, and recurrent encoders—benchmark against standard imperfect-information references (neural fictitious self-play and information-set Monte Carlo search), and probe generality on a second game, Leduc Hold'em, graded against a counterfactual-regret-optimal expert, reporting interquartile means with stratified bootstrap intervals. We release the pipeline as a game-agnostic package.

Leduc Hold'em — generality validated ✓

CFR expert is near-optimal: exploitability 2.37 → 0.026 over 400 iterations. The yardstick is monotone — mean return vs the expert: random −0.78 → CFR-10-iter −0.14 → CFR-50-iter −0.12 (weaker agents lose badly; stronger agents approach parity). So "return vs a fixed near-optimal expert" is a meaningful yardstick on a second imperfect-information game where the optimum is computable — the methodology transfers.

What we're adding (the new work)

1 · Architecture sweep

Hold the winning recipe fixed; vary only the network: MLP depth/width, Conv1D, permutation-invariant DeepSets, self-attention, and recurrent (LSTM) encoders, plus activation & regularization. Tests whether the ceiling is capacity-bound or information-bound.

2 · Real baselines

ISMCTS (information-set Monte Carlo search) graded vs the expert; NFSP cited. Blunts the #1 reject reason (weak baselines).

3 · Second game (generality)

Leduc Hold'em with a CFR-optimal expert — done: a game where the true optimum is computable, validating the expert-yardstick method.

4 · Statistical rigor

rliable: interquartile means + stratified bootstrap 95% CIs over multiple seeds. Pre-empts "Deep RL that Matters" critiques.

How we're maximizing acceptance

Apply (accepted-paper traits)

  • Explicit bulleted contributions, each traced to evidence
  • Targeted ablations that isolate which components matter
  • Statistical rigor (IQM + bootstrap CIs, multiple seeds)
  • Reproducibility: released code, configs, seeds
  • ≥1 negative result (we have several)
  • Honest scoping — "expert", not "optimal/perfect"

Avoid (rejected-paper traits)

  • Weak/missing baselines → ISMCTS + cited NFSP
  • Single-game overfit → Leduc generality + cross-recipe checks
  • No CIs / point estimates → rliable
  • Overclaiming / suggestive language
  • Not identifying the source of gains → the ablations do
  • Unclear / incremental novelty

Guzdial alignment ("How to Write an AIIDE Paper")

Intro skips "games are important," names one gap, bulleted contributions, no overclaim · related work in differentiating subsections (≥20 diverse citations) · methodology replicable by another lab, every decision justified · experiments argue appropriateness and link each contribution to evidence · results summarized + walked-through · Limitations/Future Work is defensive · conclusion = 2–4 takeaways.
